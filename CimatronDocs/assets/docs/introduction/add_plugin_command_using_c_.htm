Add a Plugin Command using C#

This section guides you through the steps needed to create a Cimatron API Application using C#. The examples here are based on Visual Studio 2019.

  1. Create C# Class library (.Net Framework) Project.ГӮВ 

    1. Create a new C# project of type Class Library (.Net Framework), a project for creating c# project library (.dll).

  2. Edit Project PropertiesГӮВ 

    1. In theГӮВ Application tab, set Target Framework to .NET Framework 4.8.ГӮВ 

      Note: If .NET Framework 4.8 is not available in the dropdown, perform following steps:

      1. Select the Install other Framework option from Target Framework dropdown. It opens a link to Download .NET SDKs for Visual Studio in your default browser (or you can directly go to the link: .NET SDKs downloads for Visual Studio (microsoft.com) )

      2. From .NET Framework section, click on the Developer Pack link for .NET Framework 4.8.

      3. If your download doesn't start automatically, click on the link to download it manually.

      4. When the download is complete, install the package.

      5. Close and reopen Visual Studio. .NET Framework 4.8 will be available in the dropdown.

    2. In Build tab, set Platform to x64 and Output path to "Cimatron Installation Directory/Cimatron/Program".

  3. Add References to the Project

    Create references to the API Interop by opening menu Project -> References and selecting all InteropГўВҖВҷs you are going to use. Attach all Cimatron API Interop files. Reference .dll location "Cimatron Installation Directory/Cimatron/Program".

    1. References Required for Plugin Command

      • CimUIInfrastructure.dll

      • interop.CimAppAccess.dll

      • interop.CimatronE.dll

    2. Optional References depending on the requirement

      • interop.CimBaseAPI.dll

      • interop.CimMdlrAPI.dll

      • interop.CIMMiscAPI.dll

      • interop.CimNcAPI.dll

      • interop.CimServicesAPI.dll

      • interop.CimElectrodeAPI.dll

  7. Register API CommandГӮВ 

    Now you are ready to use the created Plugin Command. If the command is not available, reset the toolbar, Menu, and Pane to fix the issue, see Step 8.

    1. Build the project.

    2. Open Register_API_Commands.exe from "Cimatron Installation Directory/Cimatron/Program/Register_API_Commands.exe".

    3. Click the button by the Select API Command DLL path field, and navigate to select your plug-in .dll file from the "Cimatron Installation Directory/Cimatron/Program".

    4. Select the command you want to register from the list.

    5. Click Register API Command.

  8. For Reference - Reset Toolbar, Menu, and PaneГӮВ 

    1. Close CimatronE.exe if it is open.

    2. Open the Cimatron Control Panel.

    3. In the Settings tab, click Reset All.

    4. Launch CimatronE.exe and verify the command is available at the specified locations.

This section describes the process used to create an internal plug-in command in Cimatron. Internal plug-in and plug-in commands are similar, internal plug-in commands load the commands from the InternalPlugInCommandlist.xml while the normal plug-in commands load the commands from the ExternalCommands.ini file

Step 1. Export the Visual Studio Template

  1. Open the Cimatron Control Panel.

  2. Click the Visual Studio Templates option

  3. Select Project Type: Plugin Command and export the template.

Step2. Create a Visual Studio Project for the Plugin Command

  1. Open Visual Studio.

  2. Create a new project.

  3. Choose Cimatron C# Plugin Command as the project type.ГӮВ 

  4. Click Next. The project will be created with all required dependencies and references.

Step 3: Implement the Plugin Command

Plugin Command Class

The class (such as MyPluginCommand) is derived from ICimApiCommandPlugin.

For internal commands we support Name, Caption, Tooltip, Description, and ExecuteCommand. You can set these parameters in the existing code,ГӮВ or you can use following sample code.

public class MyPluginCommand : CimUIInfrastructure.PlugIn.ICimApiCommandPlugin

{

public CimUIInfrastructure.PlugIn.ApiCommand AppendCommand()

{

CimUIInfrastructure.PlugIn.ApiCommand Command1 = new CimUIInfrastructure.PlugIn.ApiCommand();

// Set Command name

Command1.Name = "My Command";

// Set Caption string

Command1.Caption = "My Command Caption";

// Set Tool tip string

Command1.ToolTip = "My Command Tool tip";

// Set Description string

Command1.Description = "My Command Tool Description";

// Call the execute class

Command1.ExecuteCommand = new MyCommandExecution();

// Return command object

return Command1;

}

}
