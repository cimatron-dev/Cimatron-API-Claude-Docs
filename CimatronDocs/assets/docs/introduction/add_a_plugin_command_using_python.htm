Add a Plugin Command using Python

This section guides you through the steps needed to create a Cimatron API Application using Python.Ă‚Â The steps and snapshots are based on Visual Studio 2019.

Note: Python 3.7 to 3.12 are supported.

  1. Start a New Project

    1. Create a new Python Application Project.

  2. Set Publish Path

    1. In the Solution Explorer window, right-click on project name.

      Select Properties.

    2. In the Properties window, click the Publish tab.

    3. Set the Publishing Folder Location to where CimatronE.exe is present (Cimatron Installation Directory>\Cimatron\Program).

  3. Add Interop References

    Interop dlls are imported in python script using clr module. This module can be imported from pythonnet package.

    1. Click on the package manger icon next to the python environment.

      • interop.CimAppAccess.dll

      • interop.CimatronE.dll

      • interop.CimBaseAPI.dll

      • interop.CimMdlrAPI.dll

      • interop.CIMMiscAPI.dll

      • interop.CimNcAPI.dll

      • interop.CimServicesAPI.dll

      • interop.CimElectrodeAPI.dll

  6. Write Command Execution for Plugin

    1. Ensure that Interop references for required APIĂ¢Â€Â™s are added int the References.

    2. Write the code for the command in the OnCommand(self) function by referring to the code mentioned in Step 5. See the example in Appendix A.

  7. Register API Command

    1. Publish the project. Right-click on project name in the Solution Explorer and click Publish.Ă‚Â 

    2. Use a text editor such as Notepad or Notepad++ to open ExternalCommands.iniĂ‚Â from the following folder C:\ProgramData\Cimatron\Cimatron\2025.0dev\Data.

      NOTE: Navigate to this location manually rather than copying and pasting the path because it may differ depending on your version of Cimatron.

    3. Under [Plugin Ext Commands], add a new line and enter your plugin command in following format: Ă‚Â 

      PyFileName.PluginClassName = PyFileName.PluginClassName@0

      For example, using the code provided in Appendix A to create a file named PluginFile.py, the entry in ExternalCommands.ini will be PluginFile.NewPluginCommand = PluginFile.NewPluginCommand@0.

  8. Reference - Reset Toolbar, Menu, and Pane

    Perform this step only if your Command is not visible in Toolbar and Menu bar at specified locations.

    1. Close CimatronE.exe if it is open.

    2. Open the Cimatron Control Panel.

    3. In the Settings tab, click Reset All.

    4. Launch CimatronE.exe and verify the command is available at the specified locations.

Convert .py file to .pyd

  1. Create the Setup File

    Note: If setuptools package is not installed, run following command to install it; setuptools is used for packaging python projects.Ă‚Â 

    pip install setuptools

    1. Create the setup file with name `setup.py` and write following content in it; Replace PyPlugin with the name of your .py file:

      # setup.py
      Ă‚Â 
      from setuptools import setup, Extension
      Ă‚Â

      Ă‚Â  Ă‚Â  name='PyPlugin',
      Ă‚Â  Ă‚Â  ext_modules=[
      Ă‚Â  Ă‚Â  Ă‚Â  Ă‚Â  Extension(
      Ă‚Â  Ă‚Â  Ă‚Â  Ă‚Â  Ă‚Â  Ă‚Â  'PyPlugin',

      Ă‚Â  Ă‚Â  Ă‚Â  Ă‚Â  )
      Ă‚Â  Ă‚Â  ],
      )

  2. Generate C Source file

    To generate C source file, open a command window and run the following commands.Ă‚Â 

    1. cython is used to convert python script to a C extension file. If cython package is not installed, run following command to install it:

      pip install cython

    2. If terminal is open in different directory, change the directory using the following command. Replace path/to/directory with your .py file directory:

      cd path/to/directory

    3. Run following command to create C extension source file. This will generate PyPlugin.C file. Replace PyPlugin.Py with name of your plugin .py file.:

      cythonize -i PyPlugin.py

  3. Generate .pyd file

    1. Run following commands to generate .pyd file:

      python setup.py build_ext --inplace

      This will generate the .pyd file.
