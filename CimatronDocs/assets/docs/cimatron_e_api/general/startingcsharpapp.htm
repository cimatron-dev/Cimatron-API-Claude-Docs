Starting a C# Application with the CimatronAPI

As the Cimatron API uses the COM based approach, to start working with it in a C# application, you need to add references to all type libraries you going to use.

The API uses the hierarchical object model. Files are related in a parent-child manner, with each parent capable of relating to more than one child, but each child only being related to one parent. The top-level directory is the root, andÂ child objects are derived from the parent object above them in the hierarchy.

These kinds of objects are theÂ Application objects of the CimatronÂ API.

To getÂ anÂ Application objectÂ of a running active CimatronÂ application, use the AppAccess object. Its interface (IAppAccess) allows you to connect to a running active application by returning a pointer to the IApplication interface of the Application object.

If there is no running application or you want to start a new one, you have to create an instance of the Application object and receive a pointer to the IApplication interface.

When you have a pointer to the IApplication interface, you can start to execute a task of your user application.

There is no special requirement for the type of project you can create in Visual C#.
