Localization Mechanism in Standalone Application or Plugins

When developing plugins for standalone applications, it's crucial to consider localization to ensure that text can be converted into other languages. This enhances the user experience by making the application accessible to a global audience.

Localization Mechanism

Cimatron provides a robust mechanism to manage localization through its API. Users need to create their own DLL of resource files to manage localization. This approach allows Cimatron to handle localization efficiently and manage DLL loading seamlessly.

Methods for Localization

Cimatron offers two primary methods for localization:

  1. Using Localization Service
    The Localization Service is a comprehensive method for managing localization. This method is ideal for users who prefer a more integrated approach and are comfortable with COM. This interface is available from V2025.
    Detailed documentation can be found here.
  2. Using ResourceLoader
    The CommandLineParser.dll includes a class called ResourceLoader, which retrieves strings based on a given ID from the specified resource DLL. This method is faster than using the Localization Service and is suitable for users who prioritize performance. This class is available in the older version as well.

Choosing the Right Method

Both methods automatically load the resource DLL within Cimatron. However, the choice between them depends on the user's needs:

  • Localization Service: Best for users who need a comprehensive, COM-based solution.
  • CommandLineParser.dll: Preferred for users who need a faster, performance-oriented solution.

By considering these methods, developers can ensure their plugins are well-localized, enhancing usability and accessibility for users worldwide.
