Read and Write Custom Data to Cimatron Document Files (ELT) files

Users may want to read or write their own data into a Cimatron document file.

This can be done by a developer, who writes Cimatron API code based on requirement. In the Cimatron API there are two methods.

  1. Using Cimatron File Structure
    The developer must understand the file structure and add their custom information inside the elt file using Cimatron API, separating it from Cimatron internal data. This method is recommended for large custom data files. Cimatron will not read the custom data and therefore there is no performance impact when opening the file.Â 
    For more, seeÂ Using the Cimatron File Structure.

  2. Using Cimatron Doc Attribute
    The developer creates their own document attribute and attaches to the Cimatron document using Cimatron API. Using this method, a user can attach custom data inside the Cimatron. However, Cimatron must read this custom attribute while reading the file which may impact on performance when opening the file.Â 
    For more, see Using the Cimatron Doc Attribute.
